# Карточка спецификации фичи: расписание::виджет_срочные_обращения_под_календарем

1. **feature_key**: `расписание::виджет_срочные_обращения_под_календарем`
2. **Название**: Виджет «Срочные обращения» под календарем расписания (Отмены / Переносы)
   - **Конкурент**: IDENT (Версия 292)
3. **Проблема пользователя**: Когда пациент отменяет или переносит прием за 1–2 часа до начала, образовавшаяся дыра в расписании остается незанятой, если администратор немедленно не обработает отмену.
4. **Кому польза**: Главный администратор, Дежурный оператор расписания.
5. **Полное поведение**:
   - Под мини-календарем в левом сайдбаре модуля «Расписание» располагается специальный ярко-красный виджет «Срочные обращения».
   - При отмене или переносе визита кнопками из бота (Забота 2.0 / LoyalMed / Telegram) или при входящем звонке запись помещается в виджет со счетчиком минут, прошедших с момента отмены (`00:04`).
   - Клик по записи подсвечивает освободившийся слот в сетке дня и выводит список пациентов из листа ожидания с совпадающими параметрами (врач/услуга).
6. **UI-детали**:
   - Блок `Срочные отмены (N)`.
   - Кнопка быстрых действий: `Заполнить из листа ожидания` / `Снять бронь`.
7. **Данные**:
   - `urgent_cancellations` (`id`, `appointment_id`, `cancelled_at`, `reason`, `status: pending|resolved`).
8. **Наш статус**: **[НЕТ]**
   - У нас есть статус визита `cancelled` и модалка листа ожидания, но нет компактного сквозного виджета под календарем.
9. **ДЛЯ БУДУЩЕГО АГЕНТА — варианты внедрения**:
   - **Модули-кандидаты**: `apps/web/src/ScheduleView.tsx`, `apps/api/src/routes/schedule.ts`.
