# Спецификация функциональной фичи: кадры::выделенная_роль_куратор_пациентов

## 1. Канонический ключ и название
- **feature_key**: `кадры::выделенная_роль_куратор_пациентов`
- **Наименование**: Выделенная роль «Куратор пациентов» со своей аналитикой
- **Конкуренты-аналоги**: IDENT
- **Продуктовый модуль**: Кадры / CRM

---

## 2. Пользовательская проблема (User Problem)
Отсутствие автоматизации фичи "Выделенная роль «Куратор пациентов» со своей аналитикой" в клинике приводит к ручным рутинным операциям, задержкам обслуживания пациентов и риску человеческой ошибки.

---

## 3. Кому приносит пользу (Who Benefits)
Администратор клиники, Врач-стоматолог, Руководитель, Главный врач.

---

## 4. Подробный пользовательский сценарий (User Flow & Behavioral Scenario)
1. Пользователь открывает модуль "Кадры / CRM".
2. Система производит считывание текущего состояния и данных.
3. Выполняется целевое действие с отображением статуса в UI.
4. Данные синхронизируются с базой данных PostgreSQL.

---

## 5. Детали интерфейса (UI Details & Layout)
• Модальный компонент в представлении модуля Кадры / CRM.
• Кнопки управления целевым действием.
• Индикаторы статусов и апертуры предупреждений.
