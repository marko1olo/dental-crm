# Алгоритмы, Валидаторы и Общий Пакет `@dental/shared`

## 1. Пакет Общих Типов и Схем Zod (`packages/shared/src/index.ts`)

- **Валидация Документов**:
  - `documentKindSchema`: Zod-валидация всех 31 видов юридических документов.
  - `legacyTaxDeductionCertificateMinYear = 2021`, `legacyTaxDeductionCertificateMaxYear = 2023`, `taxDeductionCertificateMinYear = 2024` — временные рамки нормативных приказов ФНС для вычетов.
  - `documentFactoryGroups`: классификация документов на группы `visit`, `payment`, `tax`, `legal`, `workflow`.

- **Одонтограмма и Стоматологическая Формула**:
  - Все 32 постоянных зуба (11–48 по нотации FDI) + 20 молочных зубов (51–85).
  - Энум анатомических поверхностей зуба: `occlusal` (жевательная), `vestibular` (щечная), `lingual` (язычная), `mesial` (медиальная), `distal` (дистальная).
  - Патологии: `caries`, `pulpitis`, `periodontitis`, `filling`, `crown`, `implant`, `extracted`, `root_canal`.

## 2. Ключевые Алгоритмы Системы

### 2.1. Алгоритм 3D DICOM MPR Реконструкции КТ (`apps/web/src/mprWorker.ts` & `mprMath.ts`)
- **Вход**: Трёхмерный массив вокселей DICOM (серия КТ-срезов).
- **Логика**:
  1. Вычисление преобразования координат из мирового пространства DICOM в локальные плоскости (Аксиальная, Сагиттальная, Корональная).
  2. Интерполяция значения единиц Хаунсфилда (HU) по методу трилинейной интерполяции.
  3. Масштабирование контраста по окнам (Window Width / Window Center — Bone, Soft Tissue, Dental).
- **Выход**: Готовые каналы пикселей RGBA в Web Worker без блокировки главного UI-потока.

### 2.2. Алгоритм Нормализации Врачебной Диктовки (`apps/api/src/routes/speech.ts`)
- **Вход**: Сырой аудиопоток от микрофона браузера.
- **Логика**:
  1. Передача аудиосегмента в серверный STT-шлюз (Yandex SpeechKit / Whisper / Deepgram).
  2. Очистка хвостов и повторов через дедупликацию соседних транскриптов `appendSpeechTextWithoutDuplicateTail`.
  3. Медицинская нормализация терминов: преобразование фразы "кариес на сорок шестом дистальная" в структуру `tooth: 46, surface: 'distal', pathology: 'caries'`.
- **Выход**: Структурированный протокол приема, подставляемый в черновик ЭМК.

### 2.3. Алгоритм Расчета Обязательных Требований НДФЛ (`apps/web/src/documentLogic.ts`)
- **Вход**: Список оплаченных счетов за выбранный календарный год.
- **Логика**:
  1. Разделение сумм по кодам услуг: Код 1 (обычное лечение) и Код 2 (дорогостоящее лечение — имплантация, костная пластика, синус-лифтинг).
  2. Проверка задолженности: если услуга не оплачена в 100% объеме — сумма не включается в справку.
  3. Исключение немедицинских товаров (зубные щетки, пасты, ирригаторы из кассовых позиций).
- **Выход**: Распределенная сумма справки НДФЛ, готовая к печати и выгрузке в XML ФНС.
