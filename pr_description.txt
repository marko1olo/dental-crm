🔒 [security fix description]
🎯 What:
Fixed a Cross-Site Scripting (XSS) vulnerability in the `ConsentTemplateEditor` component when rendering the preview mode.

⚠️ Risk:
The `content` variable, populated via a `<textarea>`, was being directly set as HTML using `dangerouslySetInnerHTML`. An attacker (or a malicious user crafting a shared template) could enter raw HTML/Javascript that would be executed in the context of the user viewing the template preview, leading to a Stored or Reflected XSS attack depending on how the template is persisted.

🛡️ Solution:
Removed the usage of `dangerouslySetInnerHTML`. Instead, we now use a custom `renderPreview` function that dynamically splits the `content` string based on the known placeholder variables (like `{{patient_name}}`). The resulting chunks are mapped sequentially: placeholders are rendered as React `<strong>` elements, while all other string fragments are rendered safely as standard React text nodes. React's built-in escaping mechanisms neutralize any malicious HTML/JS while perfectly preserving the component's original functionality and appearance.
